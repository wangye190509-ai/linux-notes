##  yum/dnf

yum/dnf 是红帽系发行版（CentOS、Rocky、AlmaLinux 等）的包管理工具，用于从在线仓库下载、安装、更新、卸载软件包，并自动解决依赖。

## dnf 与 yum 的对比

- yum：RHEL 8 及之前版本使用，基于 Python 2，性能一般
- dnf：YUM 的下一代，从 Fedora 18 开始引入，RHEL 8 开始逐步取代，RHEL 9 / Rocky 9 完全淘汰 yum
  - 基于 Python 3，依赖解析更快、更高效
  - 支持模块化（modular）软件流（比如不同版本的 Python、Node.js）
  - 命令几乎和 yum 一样，**Rocky 中 yum 只是 dnf 的软连接（/usr/bin/yum → dnf-3）**

**yum 是 dnf 的软连接**

```bash
[root@localhost ~]# type yum
yum is /usr/bin/yum
[root@localhost ~]# ll /usr/bin/yum
lrwxrwxrwx. 1 root root 5 Apr 14  2024 /usr/bin/yum -> dnf-3
```

## yum常用命令

```bash
[root@localhost ~]# yum install -y       # 安装软件包
[root@localhost ~]# yum reinstall -y     # 重新安装软件包
[root@localhost ~]# yum update           # 更新所有软件包
[root@localhost ~]# yum update 软件包名   # 更新软件包
[root@localhost ~]# yum clean all        # 清空缓存及其他文件
[root@localhost ~]# yum makecache        # 重建缓存
[root@localhost ~]# yum repolist         # 查询可用的仓库
[root@localhost ~]# yum remove -y        # 删除软件包
[root@localhost ~]# yum groupremove -y   # 卸载软件包组
[root@localhost ~]# yum provides         # 查找某个命令或某个文件是由哪个软件提供的
[root@localhost ~]# yum search           # 查找软件包
[root@localhost ~]# yum list             # 列出所有可安装的软件包
[root@localhost ~]# yum list installed   # 列出所有已安装的软件包
[root@localhost ~]# yum list 软件包名称    # 列出软件包在当前可用仓库中的信息（包括已安装的和可安装的版本）
[root@localhost ~]# yum groupinfo 软件组  # 查看软件组的详细信息
[root@localhost ~]# yum history
[root@localhost ~]# yum history info 4   # 查看具体某个yum记录的详细信息

```

## 实验二：提供内网服务器软件包更新

**实验背景：**

一台server1服务器可以访问公网，一台server2服务器无法访问公网

我们可以在这台能够访问公网的服务器上下载好软件包，通过http或者ftp提供给内网中的server2机器安装软件包

**server1：**

第一步：下载软件包，可以使用`--downloadonly`选项仅下载不安装

第二步：将软件包放到合适的位置上，并且创建软件索引数据

第三步：部署ftp文件服务器，将软件包通过ftp协议共享给sever2

```bash
[root@server1 ~]# yum clean all                        # 清理缓存
[root@server1 ~]# yum update -y --downloadonly --downloaddir=/packages   # 仅下载更新包
[root@server1 ~]# ls /packages/    # 查看下载的包，看到很多.rpm包
[root@server1 ~]# mkdir /var/ftp/update
# 创建一个目录用来存放所有 RPM 包的共享文件夹。后面 vsftpd 会把这个目录共享出去，让 server2 通过 FTP 访问
[root@server1 ~]# find /packages -iname "*.rpm" -exec cp -rf {} /var/ftp/update/ \;
# 批量复制


[root@server1 ~]# yum -y install createrepo            
# createrepo用来创建软件包的索引
[root@server1 ~]# createrepo /var/ftp/update/            
# 创建软件包的索引
# 之前把所有 .rpm 包复制到了 /var/ftp/update/
# 但 yum或dnf 不认识这个目录，它需要一个 repodata 文件夹（里面包含 repomd.xml、primary.xml.gz 等索引文件）来知道有哪些包、版本、依赖关系。
# createrepo 就是自动扫描所有 rpm 包，然后生成这些索引文件。
[root@server1 ~]# ls /var/ftp/update
# 可以看见除了一堆rpm包还有一个repodata 文件夹


[root@server1 ~]# yum install -y vsftpd
[root@server1 ~]# vim /etc/vsftpd/vsftpd.conf
# 更改ftp的配置文件，允许匿名登录
anonymous_enable=yes
[root@server1 ~]# systemctl start vsftpd    # 开启vsftpd服务
```

## RPM

**RPM**（Red Hat Package Manager）是 Red Hat 系 Linux（如 Rocky Linux、CentOS、RHEL、Fedora 等）使用的软件包格式和包管理工具。

它和 Ubuntu 的 deb 包类似，但属于完全不同的生态。

### RPM 与 yum/dnf 的关系

- **rpm** 是底层工具：手动安装单个 .rpm 包，不解决依赖。
- **yum/dnf** 是上层工具：自动从仓库下载 RPM + 依赖 + 安装（日常使用）。

### RPM 常用命令速查

| 命令示例               | 作用                           | 说明 & 常见场景                                  |
| ---------------------- | ------------------------------ | ------------------------------------------------ |
| rpm -ivh xxx.rpm       | 安装 RPM 包                    | -i 安装 -v 显示过程 -h 显示进度条                |
| rpm -Uvh xxx.rpm       | 更新/升级 RPM 包               | 如果已安装就升级，没安装就安装                   |
| rpm -e xxx             | 卸载 RPM 包                    | -e erase（删除）                                 |
| rpm -qa                | grep xxx                       | 查询已安装的包（列出所有匹配的）                 |
| rpm -qi xxx            | 查看已安装包的详细信息         | 版本、描述、打包时间、依赖等                     |
| rpm -ql xxx            | 查询包安装后包含的所有文件路径 | 看包装了哪些命令、配置文件                       |
| rpm -qc xxx            | 列出这个包安装的配置文件路径   | 快速找到某个包的配置文件位置，便于修改配置或备份 |
| rpm -qf /usr/bin/httpd | 查询某个文件属于哪个已安装包   | 逆向查找“这个命令是谁装的”                       |
| rpm -qpR xxx.rpm       | 查看未安装 RPM 包的依赖        | -q 查询 -p 包文件 -R 依赖                        |
| rpm -V xxx             | 验证已安装包的文件完整性       | 检查文件是否被修改/删除                          |

###  实例：安装

用rpm安装 httpd-2.4.62-4.el9.x86_64.rpm

```bash

	system-logos-httpd is needed by httpd-2.4.62-4.el9.x86_64

# 额外安装选项

# --force 强制安装软件包（不管有没有冲突）

```

- 官方网站
  - apache: www.apache.org
  - nginx: www.nginx.org
- github
  - www.github.com

## 编译源码包所需要的环境

这取决于源码所使用的编程语言和框架等，但是Linux基础软件包一般都是由C语言编写的，所以我们使用较多的编译环境为`gcc`、`make`等等

```bash
[root@localhost ~]# yum install -y ncurses* gcc gcc-c++ make
```

## 编译

源码包编译三部曲

```bash
- 第一步：
./configure 
#可以指定安装路径，启用或者禁用功能等，最终生成makefile

- 第二步
make 
#按Makefile文件编译。可以加 -j 2 参数使用两个cpu核心进行编译

- 第三步
make install
#按Makefile定义的文件路径安装

make clean
#清除上一次make命令所产生的object文件，要重新执行
configure时，需要执行make clean。
```

## 压缩与解压缩

**打包一个文件或者目录**：

```bash
tar -cvf archive.tar /path/to/directory

-c 表示创建一个新的归档文件。
-v 表示在压缩过程中显示详细信息（verbose 详细模式）。
-f 指定归档文件的名称。

# 如果要创建gzip压缩包
tar -zcvf archive.tar.gz /path/to/directory
```

**解压缩：**

```bash
tar -xvf archive.tar

-x 表示解压缩。
-v 表示显示详细信息。
-f 指定要解压缩的归档文件。

# 如果归档文件是使用 gzip 压缩的，比如 archive.tar.gz，则需要添加 -z 选项：

tar -zxvf archive.tar.gz
```
