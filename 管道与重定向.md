# 管道与重定向

## I/O重定向

对于Linux系统，我们常见的输入输出的类型有：

- 标准输入 0
- 标准正确输出 1
- 标准错误输出 2

### File Descriptors(FD，文件描述符 )

文件描述符是操作系统内部用于管理和跟踪打开的文件和I/O资源。比如当我们打开一个文件的时候，操作系统会分配一个文件描述符给这个文件，我们可以通过这个文件描述符来对文件进行读写等操作。

| Number | Channel name | Description | Default connection | Usage        |
| ------ | ------------ | ----------- | ------------------ | ------------ |
| 0      | stdin        | 标准输入    | 键盘               | 只读         |
| 1      | stdout       | 标准输出    | 命令行             | 只写         |
| 2      | stderr       | 标准错误    | 命令行             | 只写         |
| 3+     | filename     | 其他文件    | none               | 读 and/or 写 |

```shell
# 终端1中打开vim
[root@server1 ~]# vim file.txt
# 切换到终端2查看进程
[root@server1 ~]# ps aux | grep vim
root        1523  0.0  0.2  11752  8960 pts/0    S+   13:27   0:00 vim file.txt
root        1550  0.0  0.0   3876  2048 pts/1    S+   13:29   0:00 grep --color=auto vim
[root@server1 ~]# cd /proc/1523/fd
[root@server1 fd]# ll
total 0
lrwx------ 1 root root 64 Mar 25 13:30 0 -> /dev/pts/0
lrwx------ 1 root root 64 Mar 25 13:30 1 -> /dev/pts/0
lrwx------ 1 root root 64 Mar 25 13:30 2 -> /dev/pts/0
lrwx------ 1 root root 64 Mar 25 13:30 3 -> /root/.file.txt.swp
# 这里有0 1 2 3
其中0表示标准输入
1表示标准正确输出
2表示标准错误输出
其他的都为输出到文件中
# 向文件描述符0中输入一个hello
[root@server1 fd]# echo hello > 0

# 切换到终端1查看
[root@server1 ~]# vim file.txt
hello
~ 
~
"file.txt" [New]                                                                     0,0-1         All
# 我们可以看出在linux中一切皆文件
```

### 输出重定向(覆盖,追加)

`>`：覆盖重定向

`>>`：追加重定向

`1>`：覆盖重定向正确的输出数据流

`1>>`：追加重定向正确的输出数据流

`2>`：覆盖重定向错误输出数据流

`2>>`：追加重定向错误输出数据流

`&>`：真确输出和错误输出一起重定向

`2>&1`：将错误输出重定向到正确输出

```shell
# 示例一：输出重定向(覆盖)
[root@server1 ~]# date
Wed Mar 25 13:40:39 CST 2026
[root@server1 ~]# date 1> date.txt
[root@server1 ~]# cat date.txt
Wed Mar 25 13:41:00 CST 2026
# 示例二：输出重定向(追加)
[root@server1 ~]# date 1>> date.txt
[root@server1 ~]# date 1>> date.txt
[root@server1 ~]# date 1>> date.txt
[root@server1 ~]# cat date.txt
Wed Mar 25 13:41:00 CST 2026
Wed Mar 25 13:41:26 CST 2026
Wed Mar 25 13:41:27 CST 2026
Wed Mar 25 13:41:28 CST 2026
# 示例三：错误输出重定向
[root@server1 ~]# touch file
[root@server1 ~]# cd file
-bash: cd: file: Not a directory
[root@server1 ~]# cd file 2> log.txt
[root@server1 ~]# cat log.txt
-bash: cd: file: Not a directory
# 示例四：正确和错误都输入到相同位置
[root@server1 ~]# ls /home/ /test
ls: cannot access '/test': No such file or directory
/home/:
[root@server1 ~]# ls /home/ /test &> log.txt
[root@server1 ~]# cat log.txt
ls: cannot access '/test': No such file or directory
/home/:
# 或者
[root@server1 ~]# ls /home/ /test > log.txt 2>&1
[root@server1 ~]# cat log.txt
ls: cannot access '/test': No such file or directory
/home/:
# 示例六：重定向到空设备/dev/null
[root@server1 ~]# ls /home/ /test > log.txt 2> /dev/null   # 正确的输入到log.txt错误的丢到垃圾桶里
[root@server1 ~]# cat log.txt
/home/:
```

## 进程管道

### 一、基本管道 `|`
**作用**：将前一个命令的输出作为后一个命令的输入

#### 常用案例
```bash
# 分页查看 
ll /dev/ | less

# 过滤进程
ps aux | grep 'sshd'

# 过滤软件包
rpm -qa | grep 'httpd'
yum list | grep 'httpd'

# 统计CPU前5进程
ps aux --sort=-%cpu | head -6
```

#### less 常用操作

| 按键      | 功能       |
| :-------- | :--------- |
| `空格`    | 向下翻一页 |
| `b`       | 向上翻一页 |
| `回车`    | 向下翻一行 |
| `/关键词` | 搜索关键词 |
| `q`       | 退出       |

### 二、Tee管道 `| tee`

**作用**：将输出同时显示在屏幕并保存到文件

```bash
# 排序并保存结果
du -h . | sort -hr | tee file_sizes.txt
```

### 三、关键点

| 管道类型 | 语法               | 特点                   |
| -------- | ------------------ | ---------------------- |
| 进程管道 | `cmd1 \| cmd2`     | 单向传递，只输出到屏幕 |
| tee管道  | `cmd1 \| tee file` | 既输出屏幕，又保存文件 |

**核心概念**：
- `|` 连接命令，实现数据处理链
- `tee` 分流输出，一箭双雕
